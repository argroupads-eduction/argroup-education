# Form Submission Setup Guide

## Overview
The website now has a complete form submission system with:
- ✅ Form components in hero sections (MBBS India, MBBS Abroad, MD/MS)
- ✅ Database storage via Payload CMS FormSubmissions collection
- ✅ Email notifications to argroupeduads@gmail.com
- ✅ Confirmation emails to users
- ✅ Responsive design for mobile and desktop

## Components Created

### 1. FormSubmissions Collection
**Location**: `src/collections/FormSubmissions/index.ts`
- Stores all form submissions in the database
- Fields: name, email, phone, program, message, timestamps
- Accessible in Payload CMS admin panel

### 2. HeroForm Component (Payload CMS)
**Location**: `src/components/HeroForm/index.tsx`
- Reusable form component for hero sections
- Accepts `program` prop: 'mbbs-india' | 'mbbs-abroad' | 'md-ms'
- Handles client-side validation and submission

### 3. MBBSIndiaHero Component
**Location**: `src/heros/MBBSIndiaHero/index.tsx`
- New hero type with form integrated on the right side
- Mobile responsive (form below content on mobile)
- Can be selected in Payload CMS page editor

### 4. API Endpoint
**Location**: `src/app/api/form-submission/route.ts`
- POST endpoint to handle form submissions
- Saves data to Payload CMS database
- Sends emails via Gmail
- Includes error handling and validation

### 5. FormSection Component (Marketing Site)
**Location**: `apps/frontend/components/forms/FormSection.tsx`
- Standalone form component for apps/frontend
- Can be used in any page or section

## Environment Variables Required

Add these to your `.env.local` file:

```env
# Database (required)
DATABASE_URL=postgresql://user:password@localhost:5432/ar_group_db

# Payload CMS
PAYLOAD_SECRET=your_payload_secret_here
NEXT_PUBLIC_SERVER_URL=http://localhost:3000

# Gmail Configuration (required for emails)
GMAIL_USER=argroupeduads@gmail.com
GMAIL_PASSWORD=your_app_password_here
```

### Getting Gmail App Password

1. Enable 2-Factor Authentication on your Gmail account
2. Go to: https://myaccount.google.com/apppasswords
3. Select "Mail" and "Windows Computer" (or appropriate option)
4. Copy the generated password and use it for `GMAIL_PASSWORD`

⚠️ **Important**: Never use your actual Gmail password. Always use the App Password.

## How to Use

### In Payload CMS (ar-group-education)

1. **Create/Edit a Page**:
   - Go to Pages in Payload admin
   - Edit or create a page (e.g., "MBBS in India")
   - In the Hero tab, select hero type: **"MBBS India (with Form)"**
   - Add your hero content (title, description, links, image)
   - The form will automatically appear on the right side

2. **View Form Submissions**:
   - Go to "Form Submissions" in Payload admin
   - View all submissions with timestamps
   - Each submission shows: Name, Email, Phone, Program, Message

### In Apps/Frontend

Use the FormSection component:

```tsx
import { FormSection } from '@/components/forms/FormSection'

export default function Page() {
  return (
    <section>
      {/* Your content */}
      <FormSection program="mbbs-india" title="Get Free Counselling" />
    </section>
  )
}
```

## Form Submission Flow

1. **User submits form** with name, email, phone, and optional message
2. **Frontend validates** required fields
3. **API receives** POST request at `/api/form-submission`
4. **Database saves** submission to Payload CMS
5. **Email sent** to argroupeduads@gmail.com with submission details
6. **Confirmation email** sent to user's email address
7. **Success message** displayed to user

## Troubleshooting

### Forms not sending emails?
- Check `.env.local` has `GMAIL_USER` and `GMAIL_PASSWORD`
- Verify Gmail app password is correct (not your actual password)
- Check email in Payload admin to see if submission was saved

### "Failed to submit form" error?
- Open browser console (F12) to see detailed error
- Check if API endpoint is accessible: `/api/form-submission`
- Verify `DATABASE_URL` is set correctly

### Form not appearing?
- Ensure page hero type is set to "MBBS India (with Form)" in Payload CMS
- Check if component is imported correctly in your page

## Database Schema

```sql
-- FormSubmissions table structure
CREATE TABLE form_submissions (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL,
  email VARCHAR(255) NOT NULL,
  phone VARCHAR(20) NOT NULL,
  program VARCHAR(50) NOT NULL,
  message TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
